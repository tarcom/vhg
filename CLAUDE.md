## Project overview

Website for VHG — Vester Hassing Gymnastik & Idrætsforening (a Danish sports club). The site is a static single-page application with a PHP backend for the board member portal (`/bestyrelsen/`) and a PHP calendar scraper.

Hosted on Netgiganten (`ftp.vhg.dk`) under `public_html/`. PHP 8.1+ is required (configured via cPanel handler in `.htaccess`).

## Local development

```bash
php -S localhost:8000 -t .
# or
python3 -m http.server 8000
```

No build step, no package manager. The site is plain HTML/CSS/JS.

## Deployment

**Manual FTP deploy:**
```bash
bash scripts/deploy_vhg_ftp.sh
# With explicit host/password:
FTP_PASSWORD='xxx' FTP_HOST=ftp.vhg.dk bash scripts/deploy_vhg_ftp.sh
# Dry run:
DEPLOY_DRY_RUN=1 FTP_HOST=ftp.vhg.dk bash scripts/deploy_vhg_ftp.sh
```

**GitHub Actions:** Automatically deploys to `public_html/` on every push to `main`. Requires `FTP_PASSWORD` repository secret. See `.github/workflows/deploy-ftp.yml`.

## Calendar data

The calendar page reads from a cached JSON file — it does **not** scrape live on each page load.

```bash
php scripts/scrape_conventus_calendar.php   # regenerate cache
```

Cache file: `assets/data/calendar-events.json`. Run the scraper periodically (daily/weekly) and commit or deploy the updated JSON.

## Architecture

### Frontend SPA (`js/app.js` + `index.html`)

All page content is rendered client-side via hash-based routing (`#/route`). The entire site is a single JS file (`js/app.js`) that:

- Defines navigation (`UTILITY_NAV`, `SPORT_NAV`) and all sport/page data as JS constants at the top
- Uses a `Router` that maps `#/path` patterns to render functions
- Renders sport pages, the calendar, board pages, etc. by writing HTML strings into `<main id="app">`
- Loads calendar events from `assets/data/calendar-events.json` via `fetch()`

To add or update content (board members, contact info, training times, sport descriptions), edit the data objects in `js/app.js`.

### Board portal (`bestyrelsen/`)

A simple PHP session-based auth portal. Entry point is `bestyrelsen/login.php`; `auth.php` defines all users (bcrypt hashes), helper functions, and SMTP mail. Password resets generate a random Danish word and email it via `mail()`.

User list and password hashes live in `bestyrelsen/auth.php`. Temporary password overrides are stored in `bestyrelsen/data/password_overrides.json` (gitignored).

### DNS / go-live

`www.vhg.dk` currently points to DBU's Umbraco CMS (via wildcard CNAME). To go live, add a specific `www` A-record at Netgiganten pointing to `195.242.130.12`. To roll back, delete that A-record and the wildcard takes over again.

| `js/app.js` | Entire frontend SPA — data, routing, rendering |
